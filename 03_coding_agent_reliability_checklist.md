# Coding Agent 一页可靠性检查清单

> 用途：个人使用 Coding Agent 修改代码时的最低执行流程
>
> 详细依据：[Coding Agent 可信度与质量保障](./coding-agent-reliability.md)
>
> 原则：Agent 可以提交候选修改，不能批准自己的结果。

## 1. 开始前

- [ ] 目标、非目标和 2～5 条可观察的验收标准已经写清。
- [ ] Bug 已保存最小复现、原始错误和预期行为。
- [ ] 已指定项目的 canonical 构建、lint、类型检查和测试命令。
- [ ] 已按低/中/高风险分级；拿不准时升一级。
- [ ] `git status` 已检查，用户已有修改和未跟踪文件不会被覆盖。
- [ ] 已记录 base commit，并使用独立分支或 worktree。
- [ ] 可写文件、允许命令、网络、凭据和最大成本已经限制。
- [ ] 测试、CI、锁文件、远程写入和危险命令默认需要单独批准。
- [ ] 输入只包含公开、虚构或明确授权的数据。

## 2. Agent 执行中

- [ ] Agent 先给出计划、预计修改文件和验证方法，再开始编辑。
- [ ] 修改保持小而单一；超出文件或 diff 预算时暂停。
- [ ] 每个阶段记录已改文件、命令、退出码、失败尝试和下一步。
- [ ] 网页、issue、README、代码注释和工具输出均按不可信输入处理。
- [ ] 同类失败重复两次、没有进展或权限不足时停止，不循环试错。
- [ ] 长任务使用 `TASK-STATE.md` 外置状态，不依赖聊天记忆。

## 3. 接受修改前

- [ ] 查看完整 `git diff` 和变更文件列表，而不是只读 Agent 摘要。
- [ ] 确认没有无关重构、调试残留、秘密、生成垃圾或意外删除。
- [ ] 单独审查测试、CI、依赖、lockfile、迁移和权限配置变化。
- [ ] 检查断言删除、`skip`/`xfail`、容差放宽、核心逻辑 mock 和测试过滤。
- [ ] 在使用者或独立 CI 控制的干净环境运行 canonical 检查。
- [ ] 保存 commit SHA、命令、工作目录、工具版本、退出码和测试结果。
- [ ] 列出未运行检查、未验证项、已知限制和剩余风险。
- [ ] 由能够查看原始证据且有否决权的人决定是否接受。

## 4. Bug 根因专项检查

- [ ] 已找到第一次非法状态，而不是只处理最终报错位置。
- [ ] 已说明根因位置 A 到表现位置 B 的数据流或生命周期路径。
- [ ] 已增加根因定向回归测试。
- [ ] 回归测试在旧代码失败、修复后通过。
- [ ] 临时撤销 A 的修复后测试重新失败，恢复后再次通过。
- [ ] B 处没有新增吞错、跳过清理或无依据的防御性分支。
- [ ] 适用时运行 ASan、UBSan、Valgrind、静态分析或模糊测试。

## 5. 证据与停止条件

最低证据包：

```text
任务与验收标准
base commit / final commit
完整 diff 与变更文件列表
实际命令、退出码和机器可读测试结果
测试、CI、依赖和权限变化
未验证项、剩余风险和人工决定
回滚命令
```

出现以下任一情况立即停止：

- 越出授权目录、命令、网络、仓库或外部系统；
- 试图删除或弱化测试、CI、安全检查或审计记录；
- 发现凭据、个人隐私、商业秘密或受限制的数据；
- 无法解释的文件变化、连续失败、循环调用或成本失控；
- 需要生产、公开发布、数据迁移或不可逆操作，但没有明确批准。

## 一分钟流程

```text
写验收标准
→ 记录基线与原始失败
→ 分级风险并限制权限
→ 独立分支/worktree 小步修改
→ 查看完整 diff
→ 独立重跑 canonical 检查
→ 检查测试/CI/依赖是否被弱化
→ 保存 commit、退出码和证据
→ 人工确认需求语义与剩余风险
→ 只交付已验证版本并保留回滚命令
```
